Core Architectural Pillars

Zero-Trust Security Model

Utilizes student-scoped ephemeral tokens to ensure non-repudiation and strict access control. The middleware never stores student passwords.

Idempotent Transaction Layer

Prevents double-submission anomalies through unique transaction identifiers and state locking. Once a file is verified, re-submissions are rejected.

Asynchronous Orchestration

Decouples high-volume physical scanning from Moodle's synchronous API limits. Includes Scenario B Mitigation (Circuit Breaker) for maintenance windows.
